## Reporting a Vulnerability

If you discover a security vulnerability, please report it responsibly.

DO NOT open a public issue.

Instead:

1.  Contact the maintainer privately
2.  Provide detailed information
3.  Include reproduction steps if possible

Recommended information:

-   Description of the vulnerability
-   Steps to reproduce
-   Impact level
-   Suggested fix (if available)

------------------------------------------------------------------------

## Response Time

We aim to respond within:

-   Initial response: 72 hours
-   Investigation: as soon as possible
-   Fix release: depending on severity

------------------------------------------------------------------------

## Disclosure Policy

We follow responsible disclosure:

1.  Issue is reported privately
2.  Maintainers investigate
3.  Fix is prepared
4.  Public disclosure after fix release

------------------------------------------------------------------------

## Security Best Practices for Users

-   Run the latest version
-   Avoid modifying executable behavior
-   Do not disable system protections
-   Review blocked application lists carefully

------------------------------------------------------------------------

## Thank You

Responsible security reporting helps keep users safe.
